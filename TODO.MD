## Fase 47: Hierarquia de Roles вЂ” Dev (supremo) vs Admin (restrito) вњ… (v2.6.17)

### Contexto

A soluГ§ГЈo Г© quebrar em 3 roles com responsabilidades distintas:

| Role | Acesso a dados prГіprios | Acesso a dados alheios | Gerenciar usuГЎrios | Export total | Deletar usuГЎrios |

**PrincГ­pio:** `admin` gerencia usuГЎrios (promove/rebaixa entre admin/usuario) e exporta dados totais, mas no CRUD de dados (cilindro, elemento, leitura, pressao, amostra) sГі enxerga os prГіprios вЂ” como `usuario`. `dev` substitui o poder absoluto que `admin` tinha antes, incluindo deletar qualquer dado ou conta.
